Parch Platform Overview

Call stack, chat stack, and host stack

The platform is now split into focused services. This relay keeps call_app authentication, signaling, billing, and the landing pages. Web chat and host-oriented APIs run on the dedicated chat relay service, with a Linux host runtime for spaces and channel state.

relay_server

Call App Core

This service handles call authentication, call room signaling, and billing workflows.

  • WS signaling: /ws
  • Auth: /call/register, /call/login
  • Billing/account: /call/create-checkout-session, /call/api/account
chat_relay

Web Chat Service

Public-key web chat now runs on a dedicated service and domain for cleaner separation.

  • Domain: chat.parchchat.com
  • Client app: /client
  • Identity: public key with local browser key storage + import/export
  • Plain-language guide: /chat/how-it-works
host_client

Linux Host Runtime

The host application is now a Linux service with migration-free schema bootstrap.

  • Service: parch-host
  • No desktop GUI dependency
  • Official host can auto-seed and auto-invite official community space

Current Routing

  • parchchat.com serves landing pages and call_app endpoints from this relay.
  • parchchat.com/client redirects to chat.parchchat.com/client.
  • sfu.parchchat.com stays dedicated to SFU traffic with existing auth checks.
